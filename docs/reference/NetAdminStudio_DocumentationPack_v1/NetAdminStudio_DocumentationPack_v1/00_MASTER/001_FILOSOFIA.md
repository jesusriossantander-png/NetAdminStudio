# Filosofía del Producto

## Filosofía

NetAdmin Studio debe sentirse como una herramienta profesional, no como un conjunto de scripts.

## Principios de experiencia

1. Mostrar estado antes de mostrar acciones.
2. Explicar riesgos antes de aplicar cambios.
3. Permitir deshacer cuando sea posible.
4. Registrar cada modificación.
5. Usar lenguaje claro, pero permitir ver detalles técnicos.
6. No saturar con comandos.
7. Dar recomendaciones inteligentes.
8. Priorizar estructura visual limpia.

## Frase interna

> El administrador nunca debería levantarse de su escritorio para hacer algo que pueda resolverse de forma segura desde la red.

## Modo técnico

Toda acción importante debe poder mostrar:
- Qué comando se ejecutará.
- Qué servicio o API se usará.
- Qué permisos requiere.
- Qué resultado obtuvo.
