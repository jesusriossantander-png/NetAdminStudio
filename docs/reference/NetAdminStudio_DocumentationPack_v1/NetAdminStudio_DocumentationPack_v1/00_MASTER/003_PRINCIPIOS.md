# Principios Fundamentales

## Principios

### 1. Diagnosticar antes de modificar
Toda pantalla debe mostrar estado actual antes de permitir cambios.

### 2. Seguridad por defecto
Ningún permiso peligroso debe aplicarse sin confirmación.

### 3. Historial permanente
Los cambios importantes deben generar eventos históricos.

### 4. Agente liviano
El Agent debe consumir pocos recursos y no molestar al usuario.

### 5. Sin dependencia de Internet
La red local debe ser suficiente para operar.

### 6. Modularidad
Cada módulo debe poder crecer sin romper otros.

### 7. Transparencia
El usuario administrador debe poder ver qué se está haciendo.

### 8. Reversibilidad
Si se modifican permisos o configuración crítica, debe existir backup previo.

### 9. Escalabilidad
Diseñar para 5 PCs, pero preparar para 500.

### 10. Producto, no script
Cada función debe integrarse en UX, logs, seguridad y auditoría.
