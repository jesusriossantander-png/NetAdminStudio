# NetAdminStudio — Pack 4B
## Auditoría, Reportes, Historial y Cumplimiento

Este pack define la arquitectura funcional, técnica y visual para centralizar:

- eventos de auditoría;
- historial de cambios;
- trazabilidad de acciones;
- reportes operativos y ejecutivos;
- exportaciones;
- evidencia;
- retención;
- cumplimiento;
- revisiones periódicas;
- indicadores;
- alertas de control.

## Principios

1. Toda acción relevante debe dejar evidencia.
2. La auditoría es inmutable.
3. Los reportes se construyen sobre datos trazables.
4. Cada dato debe conservar origen, fecha y confianza.
5. La retención debe ser configurable.
6. No se debe borrar evidencia crítica desde la UI.
7. Las exportaciones deben ser reproducibles.
8. Los permisos de lectura de auditoría son independientes de los permisos operativos.
9. Los reportes sensibles requieren control de acceso.
10. La evidencia debe poder correlacionarse con usuario, agente, activo, job y cambio.
