# Checklist de aceptación

- [ ] Los eventos son append-only.
- [ ] Cada evento tiene hash.
- [ ] Existe correlationId.
- [ ] La auditoría se puede buscar y filtrar.
- [ ] Los reportes son reproducibles.
- [ ] Cada reporte conserva versión y parámetros.
- [ ] Los reportes sensibles requieren permiso.
- [ ] El cumplimiento conserva evidencia.
- [ ] Las reglas están versionadas.
- [ ] La retención es configurable.
- [ ] Las purgas quedan auditadas.
- [ ] Existe verificación de integridad.
- [ ] No hay SQL libre en la UI.
