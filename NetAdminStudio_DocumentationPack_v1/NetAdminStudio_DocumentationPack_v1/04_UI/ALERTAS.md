# Alertas e Incidentes

## Estilo UI

- Menú lateral.
- Encabezado claro por módulo.
- Tarjetas de estado.
- Tabla central.
- Panel derecho de detalle.
- Botones principales arriba.
- Estados: verde correcto, amarillo advertencia, rojo error, azul acción.

## Regla UX

Primero diagnóstico, luego acción.

## Alertas

- Spooler detenido.
- Disco crítico.
- PC offline.
- Agent desconectado.
- Recurso compartido inseguro.
- Usuario sin contraseña.
- Impresora fuera de línea.

## Incidentes

Cada alerta puede convertirse en incidente.
Incidente tiene:
- Estado.
- Evidencias.
- Causa probable.
- Acción sugerida.
- Historial.
